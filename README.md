# 🏡 시놀로지(Synology) NAS 공부 스케줄러 배포 가이드

본 애플리케이션은 가족 모두가 스마트폰, 태블릿, PC, 스마트 TV 등으로 접속해 자녀의 공부 계획을 세우고 실시간 완료 스탬프를 찍어 기록을 공유할 수 있는 홈 플래너 웹 서비스입니다. 

두 가지 방식 중 본인의 시놀로지 설정 환경에 적합한 방식을 선택해 구동해주세요.

---

## 📂 프로젝트 파일 구성
NAS에 올리기 전, 전체 파일들이 한 폴더 안에 들어있어야 합니다.
- `index.html` : 메인 뷰어 및 To-Do 체크 화면 (아이용)
- `admin.html` : 스케줄 수정, 히스토리 조회, 전체 환경 설정 (부모용)
- `style.css` : 공용 프리미엄 글래스모피즘 CSS 스타일시트
- `app.js` : 메인 화면 동작 및 통계 제어 스크립트
- `admin.js` : 부모 관리화면 제어 스크립트
- `data.json` : 학습 스케줄 및 완료 기록이 저장되는 파일
- `api.php` : Web Station(PHP) 배포용 데이터 제어 API
- `server.js` : Docker / Node.js 배포용 서버 스크립트
- `package.json` : Node.js 의존성 관리 파일

## 🛠️ 배포 방식 A: Web Station + PHP 이용 (가장 추천 & 초간단)

도커(Docker) 컨테이너 설치 없이 시놀로지의 기본 웹 호스팅 기능(Web Station)을 사용하여 5분 만에 배포하는 방식입니다.

### 1단계: 시놀로지 패키지 설치
1. 시놀로지 DSM에 로그인 후 **패키지 센터**를 엽니다.
2. `Web Station` 패키지를 검색해 설치합니다.
3. `PHP` 패키지(예: **PHP 8.0, 8.2** 혹은 그 이상)를 설치합니다.

### 2단계: 파일 업로드
1. **File Station**을 엽니다.
2. Web Station이 설치되면 자동으로 생성되는 `web` 공유 폴더로 이동합니다.
3. `web` 폴더 안에 새 폴더를 만들고 이름을 `scheduler`로 지정합니다. (예: `/web/scheduler`)
4. PC에 다운로드한 프로그램 파일들(**전부**)을 `scheduler` 폴더 안으로 업로드합니다.

### 3단계: ⚠️ 파일/폴더 쓰기 권한 설정 (매우 중요)
웹 브라우저에서 체크 표시를 누르면 `data.json` 파일을 수정해야 하므로, NAS 웹 서비스 그룹에 쓰기 권한을 주어야 합니다.
1. **File Station**에서 복사한 `scheduler` 폴더(혹은 내부의 `data.json` 파일)를 마우스 우클릭한 뒤 **속성**을 누릅니다.
2. 상단 탭에서 **권한**을 선택합니다.
3. **생성** 버튼을 누릅니다.
4. 사용자 또는 그룹 목록에서 **`http`** 또는 **`system-service-webstation`** (DSM 버전에 따라 상이) 그룹을 찾습니다.
5. 권한 항목 중 **읽기**와 **쓰기** 체크박스를 모두 선택한 뒤 **저장**을 누릅니다.
6. 하단의 **"이 폴더, 하위 폴더 및 파일에 적용"**에 체크하고 최종 적용합니다.

### 4단계: 접속하기
- 같은 공유기(Wi-Fi) 망에 연결된 기기에서 다음 주소로 접속합니다.
  - 주소: `http://[시놀로지_NAS_아이피]/scheduler/index.html`
  - (예시: `http://192.168.0.100/scheduler/index.html`)

## 🐳 배포 방식 B: Docker (Container Manager) 이용

도커 컨테이너 환경에서 독립적으로 Node.js 백엔드 서버를 띄워 포트 단위로 접속하는 방식입니다.

### 1단계: 파일 업로드
1. **File Station**을 열어 `docker` 공유 폴더로 이동합니다.
2. `docker` 폴더 하위에 `study-scheduler` 폴더를 만듭니다.
3. 해당 폴더에 모든 프로젝트 파일들을 업로드합니다.

### 2단계: 컨테이너 생성 및 구동 (Container Manager)
1. 시놀로지 **Container Manager** (구명칭 Docker) 패키지를 엽니다.
2. **프로젝트**(Project) 탭을 눌러 **생성**을 클릭합니다.
3. 프로젝트 이름에 `study-scheduler`를 적어줍니다.
4. 경로를 아까 파일을 업로드한 `/docker/study-scheduler`로 선택합니다.
5. 소스 구성에서 **`docker-compose.yml`** 작성을 선택하고 아래 내용을 붙여넣습니다:

```yaml
version: '3'
services:
  app:
    image: node:18-alpine
    container_name: study-scheduler
    working_dir: /app
    volumes:
      - .:/app
    ports:
      - "3000:3000"
    command: sh -c "npm install && npm start"
    restart: always
```

6. 단계를 진행해 프로젝트를 완료하고 구동합니다.
7. 최초 실행 시 컨테이너 내부에서 패키지 다운로드(`npm install`)가 진행되며 잠시 후 서버가 가동됩니다.

### 3단계: 접속하기
- 주소: `http://[시놀로지_NAS_아이피]:3000`
- 포트 `3000`을 통해 접속할 수 있습니다. (시놀로지 방화벽에서 3000 포트 허용 여부를 체크해 주세요)

## 🔑 초기 패스워드 및 사용 방법

### 1. 기본 설정 정보
- **기본 아이 이름**: 아름이
- **부모 설정 PIN 번호**: `1234` (비밀번호)
  - 부모 설정 페이지(`admin.html`)로 이동할 때 이 PIN 번호를 입력해야 잠금이 해제됩니다.
  - 잠금 해제 후 **스케줄러 환경 설정** 탭에서 **자녀 이름**, **새로운 PIN 번호**, **동기부여 칭찬 문구**들을 마음대로 수정할 수 있습니다.

### 2. 데이터 백업 및 복원
- 컴퓨터를 바꾸거나 NAS를 포맷하더라도 공부 데이터를 잃지 않도록, 부모 설정 화면 하단에 **"백업용 JSON 다운로드"** 버튼을 제공합니다.
- 백업 파일을 다운로드해 둔 다음, 나중에 **"백업 JSON 파일 선택"**을 통해 클릭 몇 번만으로 이전 공부 스케줄과 아이가 채운 완료 히스토리 스탬프판 전체를 복구할 수 있습니다.
